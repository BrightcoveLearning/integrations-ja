    アドビ-AEM-Brightcove ブコネクター:設定

    このトピックでは、Adobe エクスペリエンスマネージャー（AEM）用の Brightcove コネクタの設定について説明します。

    はじめに

    Adobe-AEM Brightcove コネクタを使用すると、AEM 内で Brightcove ビデオクラウドの動画とプレーヤーを管理し、AEM ページに動画を簡単に埋め込むことができます。

    このガイドでは AEM 6.3 を使用していることに注意してください。コネクタは AEM 6.2  および  AEM 6.4 でも動作し、手順は似ています。

    設定手順

    以下のセクションでは、クラシックインターフェイスを使用して Adobe-AEM BrightCove コネクタを構成する手順について説明します。これらの手順を完了する前に、コネクタをインストールする必要があります。「 Adobe-AEM-BrightCove コネクタ」を参照してください。インストール

    DAM トランスコーディングを無効にする

    デフォルトでは、AEM DAM にビデオアセットをアップロードすると、バックグラウンドで ffmpeg が実行され、ビデオをさまざまなフォーマットにトランスコードします。  DAM にトランスコードされたアセットが不要な場合は、トランスコーディングをオフにして、処理時間とストレージを節約する必要があります。

    1. AEM ツールを開きます。
    2. ワークフローを選択:
    3. モデルの選択:
    4. リストビューの選択:
    5. DAM 資産の更新を選択します。
    6. [編集] を選択します。
    7. ワークフローから [FFmpeg トランスコーディング] をダブルクリックして編集します。
    8. [ステッププロパティ] ダイアログで [プロセス] タブを選択します。
    9. プロセスを [ 操作なし ] に変更します。
    10. OK をクリックしてダイアログを閉じ、変更を保存します。

    ローカルでトランスコードされたビデオアセットが必要な場合は、AEM サーバーに ffmpeg がインストールされていることを確認する必要があります。

    1. コマンドラインを開いて次のように入力して、 FFmpeg がインストールされていることを確認します。 
      ffmpeg
    2. ffmpegコマンドが見つからない場合は、インストールする必要があります。
      • Mac にインストールする
      • Windowsにインストールする
      • Linux にインストールする

    アカウント情報を入力してください

    これで、Video Cloud アカウント情報を使用して AEM BrightCove コネクタを設定する準備ができました。

    1. AEM Web コンソール設定マネージャーを開きます（http://localhost:4502/system/console/configMgr）
    2. Brightcove サービスを検索して、サービスを見つけます。
    3. +サインをクリックして、新しい設定を追加します。
    4. ダイアログでは、下図のように値を入力します。クライアント ID とクライアントシークレットの取得方法については、「 API 認証資格情報の管理」を参照してください。ここで必要な権限は次のとおりです。
      • 設定フォームの推奨値は次のとおりです。
        • アカウントの別名 : コネクタに表示するアカウントの名前を入力します (必須)
        • アカウント ID : Brightcove ブのパブリッシャー ID を入力します (必須)
        • クライアント ID : Brightcove ブ API 認証ページからBrightcove ブのクライアント ID を入力します (必須)
        • クライアントシークレット : Brightcove ブのクライアントシークレット API を入力します (必須)
        • [ 許可されたグループ ]: コネクタにアクセスするグループを指定します (必須)。そのグループが、含まれているグループであることを確認してください。
        • Players Store Path : この Brightcove アカウントのプレーヤーには、デフォルト値を維持するか、 選択したフォルダを使用します (必須)
        • デフォルトの動画プレーヤー ID : デフォルト値をそのまま使用するか、別のプレーヤー ID を選択します (必須)
        • 既定のビデオプレーヤーキー : 非推奨、空白のままにするか、以前のコネクタの値を使用します
        • デフォルトのプレイリストプレーヤー ID : プレイリストを使用する予定の場合は、プレイリストが有効になっているプレーヤー ID を入力します (必須)
        • デフォルトのプレイリストプレーヤーキー : 非推奨、空白のままにするか、以前のコネクタの値を使用します
        • プロキシサーバー : 空白のままにするか、以前にプロキシを使用していた場合は proxy を入力します。
        • ダム統合パス : この Brightcove アカウントの動画アセットにデフォルト値を維持するか、選択したフォルダを使用します（必須）
        • インジェストプロファイル : アップロードされた動画に使用する Brightcove インジェストプロファイルを指定します。空白の値はデフォルトプロファイルを使用します。空白/デフォルトが機能しない場合は、Brightcove サポートに Video Cloud バックエンドでデフォルトプロファイルが定義されているかどうか確認してください
    5. 設定をテストします。  http://localhost:4502/brightcove/admin を参照し、右上隅のプルダウンに追加したばかりのBrightcove ブのアカウントを選択します。 動画が表示されない場合は、次のいずれかの原因が考えられます。
      1. API資格情報 アカウントID、クライアントID、およびクライアントシークレット がコネクタ構成に正しく入力されていない可能性があります
      2. Brightcove ブで API 認証情報が適切に設定されていません
      3. 現在ログインしているユーザーは、コネクタ構成の「許可されたグループ」 のいずれかのメンバーではありません
      4. ネットワーク設定により、Brightcove ブのクラウドサーバへの API アクセスがブロックされています。詳細は、「ネットワークおよび API 接続に関する問題のデバッグ」を参照してください。
    6. 複数の Video Cloud アカウントを使用する場合は、アカウントごとに前の手順を繰り返します。

    Brightcove ブプレーヤーをセットアップする

    最後の設定タスクは、Brightcove プレーヤーをセットアップすることです。

    1. サイト管理ツール（クラシック UI では http://localhost:4502/siteadmin#/content、タッチ UI では http://localhost:4502/sites.html/content）に移動します。
    2. 最上位のWebサイトフォルダーに、構成内の プレーヤーストアパス に一致する新しいフォルダーを作成します。 デフォルトでは、 設定内のフォルダ名は、 呼ばれるプレーヤー内にフォルダを持つ、Brightcove ブツールと呼ばれます。 複数の Brightcove アカウントに接続している場合は、アカウントごとに一意のプレーヤーフォルダを作成できます。
    3. プレーヤーフォルダ内に新しい Brightcove プレーヤーページを作成します。
    4. ページを作成したら、そのページを右クリックし、クラシック UI のメニューから [プロパティ] を選択します。タッチ UI でページを選択し、[プロパティ] をクリックします。
    5. ダイアログの [Brightcove Basic] タブと Brightcove アカウントに移動し、リストからプレーヤーを選択します。
    6. 複数のプレーヤーを公開できるようにする場合は、上記の手順を繰り返して、各プレイヤー用のページを作成します。
    7. プレイリストも使用する場合は、上記の手順を繰り返して、プレイリストプレーヤー用のページを作成します。
    8. 複数の Brightcove アカウントを使用している場合は、アカウントごとにプレーヤーページを作成する必要があります。

    Brightcove アセットロケーションのメタデータスキーマをアクティブ化

    v5.4 コネクタ以降で Touch UI を使用している場合は、Brightcove アセットロケーションのメタデータスキーマを有効にする必要があります。  これは、クラシック UI のみを使用している場合は必要ありません。 次の手順に従って、メタデータスキーマをアクティブ化します。

    1. AEM ツールを開きます。
    2. アセットの選択:
    3. メタデータスキーマの選択:
    4. [Brightcove ブ] を選択し、[フォルダに適用] をクリックします。
    5. brightcove_assets フォルダが存在しない場合は、brightcove_assets という名前のフォルダを作成します。
    6. 「brightcove_assets」という名前を入力してください:
    7. brightcove_assets を選択し、適用をクリックします。

    これで、AEM-BrightCove コネクタの設定が完了しました。「コネクタの使用」に進みます。
