    アドビ-AEM-Brightcove ブコネクター:コネクタの使用

    このトピックでは、AEM で動画を管理および公開するための Adobe-AEM BrightCove コネクタ 5.5.x の使用方法について説明します。

    はじめに

    Adobe-AEM Brightcove コネクタを使用すると、AEM 内で Brightcove ビデオクラウドの動画とプレーヤーを管理し、AEM ページに動画を簡単に埋め込むことができます。

    このガイドでは AEM 6.3 を使用していることに注意してください。コネクタは AEM 6.2  および  AEM 6.4 でも動作し、手順は似ています。

    Brightcove ブ AEM コネクタは、次のコンポーネントで構成されています。

    • ビデオプレーヤーの公開
    • ビデオコンテンツ管理
    • Brightcove ブの管理

    ビデオプレーヤーの公開

    動画プレーヤーを公開すると、ユーザーは Brightcove Player コンポーネントを Web ページにドラッグし、そのコンポーネントの Brightcove アカウント、動画、およびプレーヤーを選択できます。このコンポーネントは、クラシック UI モードの v4.2 コネクタでのみ使用可能です。v5.1 コネクタでは、このコンポーネントは、クラシック UI モードとタッチ UI モードの両方で使用できます。v4.2 からのもう 1 つの変更点として、従来の Smartplayer コンポーネントは使用できなくなっています。現在は HTML5 Brightcove ブプレーヤーコンポーネントのみが使用可能です。従来の SmartPlayer コンポーネントを含む既存の Web ページには、これらのコンポーネントが含まれている可能性がありますが、新しい SmartPlayer コンポーネントを Web ページに追加することはできません。

    1. 次のようなサイトページを参照します。/editor.html/content/we-retail/us/en.html

    このサイトページの Brightcove Player コンポーネントを有効にします。

    1. [テンプレートを編集] ダイアログを開きます。
    2. 右上隅の [構造] メニューから、[構造] を選択します。
    3. [ コンポーネントをここにドラッグ ] 領域の [レイアウトコンテナ] をクリックし、上部にメニューを表示し、[ポリシー] アイコンをクリックします。
    4. [許可されるコンポーネント] リストで Brightcove を検索し、[Brightcove 動画プレーヤー] および [プレイリストプレーヤー] コンポーネントを選択します。
    5. アイコンをクリックして変更を保存します。
    6. 左側のコンポーネントリストからテンプレートに Brightcove プレーヤーをドラッグできるようになりました。
    7. 埋め込まれた Player コンポーネントを選択し、[ 構成 ] アイコンをクリックします。
    8. [構成] ダイアログで、アカウント、ビデオ、およびプレーヤーを選択し、以下をクリックします。
    9. 場合によっては、Brightcove プレーヤーの詳細オプションを設定する必要がある場合があります。 最も一般的な状況は、Brightcove ブのプレイリストプレーヤーを使用している場合、プレイリストのルックアンドフィールを設定するために CSS を追加する必要があることです。

      以下は、プレイリストをフォーマットするためのサンプルCSSです。サムネイルの高さと幅を計算する必要があることに注意してください。開始点として、以下の数式を使用できます。

      THUMBNAILWIDTH = Round((PLAYERWIDTH - NUMTHUMBNAILS * 4)/ NUMTHUMBNAILS);
      THUMBNAILHEIGHT = Round(THUMBNAILWIDTH * PLAYERHEIGHT / PLAYERWIDTH);

      CSSの例:

      
        <style type="text/css">

  .vjs-up-next-text {

    /*display: none;*/
    opacity: 1;

    text-align: right;
  }

    10. 最初に、ビデオが再生できないことを示すエラーが表示されることがあります。これは、プレーヤーが編集モードで正常に機能しないためです。 プレビューモードに切り替えて、ビデオを再生します。

    ビデオコンテンツ管理

    v5.x Connector の動画コンテンツ管理では、AEM DAM  を使用してビデオをBrightcove ブにアップロードし、Brightcove ブのメタデータを更新します。コネクタの以前のバージョンでは、の管理ページを使用して Brightcove に動画をアップロードしていました。/brightcove/admin

    AEM DAM セットアップ

    AEM DAM をBrightcove ブで初めて使用する前に、Brightcove ブの設定ページで指定された DAM 統合パスフォルダ構造を作成する必要があります。/system/console/configMgrデフォルトは、/content/dam/brightcove_assets設定ページで変更しない場合です。 このフォルダの下には、Brightcove ブアカウントごとにフォルダを作成する必要があります。フォルダ名は、Brightcove ブ設定の数値アカウント ID で、フォルダタイトルは Brightcove 設定のアカウントエイリアスである必要があります。

    クラシック UI では、フォルダはに作成できます/damadmin#/content/dam。タッチUIでは、フォルダはに作成されます/assets.html/content/dam。または、の管理ページの [ SYNC DATABASE ] /brightcove/adminボタンにより、設定済みの Brightcove アカウントごとに必要なフォルダが AEM DAM 内に作成され、Brightcove ブアカウントにすでに存在するすべての動画がインポートされます。Brightcove ブ内の一部またはすべての動画を同期したくない場合、 Brightcove ブタグが AEM NO_DAM の動画は AEM DAM に同期されません。

    また、クラシック UI で AEM DAM を使用している場合は、表示時に次に示すように [タイトル] フィールドを有効にします/damadmin#/content/dam。

    動画を AEM DAM にアップロードしてからBrightcove ブに公開すると、 元の動画ファイルは AEM DAM に保持されます。実装によっては、AEM データベースに「master」ビデオを保持することをお勧めしますが、他の実装では AEM データベースのサイズが大きくなりすぎるので好ましいものではありません。AEM DAM に元の動画を保存したくないが、Brightcove Studio インターフェイスを使用して動画をアップロードしたくない場合は、 Brightcove ブに一度公開した後、AEM DAM 内の動画を削除するだけで、サムネイル画像を使用してBrightcove ブからインポートし直すことができます。プレースホルダーは、同期データベース操作によってトリガーされ、次の同期中に。

    AEM DAM 経由でビデオをアップロードし、Brightcove ブに公開してから、後でBrightcove ブ Studio  でその動画のソースファイルを置き換えると、次の SYNC データベース中に AEM DAM に新しいアセットが作成されます。 オペレーションはソースファイルを置き換えたときにBrightcove ブで新しい VideoID が作成されるためです。 ただし、AEM DAM から公開された動画のメタデータをBrightcove ブ Studio で更新することはできます。これにより、AEM   DAM 内の既存のアセットのメタデータが期待どおりに更新されます。

    クラシック UI で新しい動画をアップロードする

    1. クラシック UI で新しい動画をアップロードするには、を参照して/damadmin#/content/dam/brightcove_assets、使用する Brightcove アカウントのフォルダを開き、[ > 新規新規ファイル ] をクリックします。
    2. アップロード後、[タイトル] フィールドを編集します。空白のままにすると、アセットをBrightcove ブにアップロードするアセットを公開するときに DAM  にアップロードされた動画ファイルの名前に設定されます。[タイトル] フィールドは、新しい資産の [タイトル] フィールドをクリックすると、DAM 資産リストビューで編集できます。
    3. または、新しいアセットを開いて [ タイトル ] フィールドを編集し、変更を保存することもできます。
    4. その他のメタデータフィールドはオプションです。ただし、Brightcove ブのカスタムフィールドの場合もあります。ただし、AEM Classic UI の制限により、5.x バージョンの Connector でサポートが追加された Brightcove カスタムフィールドは、クラシック UI では編集できず、代わりにタッチ UI を使用する必要があります。
    5. この時点で、ビデオは AEM DAM にのみ存在します。動画を Brightcove と同期するには、アセットリストビューで新しいアセットを選択し、[アクティブ化] をクリックして Brightcove に動画を公開します。

    タッチ UI でのファイルのアップロード

    1. Touch UI で新しい動画をアップロードするには、参照して/assets.html/content/dam/brightcove_assets/、使用する Brightcove アカウントのフォルダを開き、[ > ファイルの作成 ] をクリックします。
    2. ビデオをアップロードしたら、必要に応じて [ タイトル ] フィールドを編集します。[タイトル] フィールドを編集するには、新しいアセットを開き、[ プロパティを表示] を選択し、[基本] プロパティページで [タイトル] を編集します。
    3. その他のメタデータフィールドはオプションです。ただし、Brightcove ブのカスタムフィールドの場合もあります。
    4. 動画を Brightcove と同期するには、アセットビュー内で [公開] を選択します。
    5. または、リスト、カード、列ビューで新しいアセットを選択し、[公開] をクリックして Brightcove に動画を公開することもできます。

    AEM DAM でのサムネイルとポスター画像の変更

    DAM の動画アセットにある brc_thumbnail.png と brc_poster.png という名前のレンディションを上書きすることで、Brightcove ブでサムネイル画像とポスター画像を更新できます。クラシック UI で、/damadmin #/content/dam/brightcove_assets を参照してビデオアセットを開き、[レンディション] タブの [アップロード] をクリックします。

    brcthumbnail.pngまたはbrcposter.pngをアップロードした後、アセットを保存およびアクティブ化して、新しいサムネイルおよびポスター画像をBrightcoveにアップロードします。Touch UI で、/assets.html/content/dam/brightcove_assets/ビデオアセットを参照して開きます。以下に示すように、レンディションをクリックします。

    次に [レンディションを追加] をクリックし、brc_thumbnail.png または brc_poster.png という名前のファイルをアップロードし、アセットを公開して、新しいサムネイルとポスター画像をBrightcove ブにアップロードします。

    AEM  からBrightcove ブへのビデオのインポート

    新しい動画がBrightcove ブ Studio に直接アップロードされる場合、またはメタデータがBrightcove ブ Studio で直接編集されている場合、 Brightcove ブの管理ページの [ 同期データベース ] ボタンを使用して/brightcove/admin新しい動画や更新されたメタデータをBrightcove ブから AEM DAM に同期します。  Brightcove ブから DAM にインポートされた動画アセットは、元の動画ファイルを AEM DAM にコピーするのではなく、アセットのプレースホルダーとして Brightcove サムネイルを使用します。したがって、Brightcove ブから動画をインポートしても、AEM データベースのサイズは大幅に増加しません。

    データベース同期は、スケジュールに従って実行するようにセットアップすることもできます。で設定コンソールを開き、「Brightcove アセットインテグレーション Cronjob スケジューラ設定」を探して開きます。/system/console/configMgr

    「CRON 有効化」をクリックし、「CRON スケジューラ」フィールドに有効な CRON 式を入力します。

    次のコマンドは、OS から手動で使用して、同期プロセスをトリガーすることもできます。

    /usr/bin/curl —anyauth < —ユーザーユーザー名 >: < パスワード > —サイレント — < 圧縮されたインスタンス > /bin/brightcove/dataload.html

    <USERNAME >: < PASSWORD は、Brightcove ブのアカウントへのアクセス権とタグとコンテンツを作成する権限を持つ有効な AEM > ユーザーである必要があります。

    Brightcove ブの管理

    Connector の以前のバージョンでは、のBrightcove ブ管理ページを使用して、Brightcove ブに動画をアップロードし、それらの動画のメタデータを編集していました。/brightcove/adminAEM DAM の新しいサポートにより、Brightcove Admin ページは通常、Brightcove ブアカウントのデータのみを表示するために使用されました。たとえば、Brightcove ブの管理者ページには、Brightcove ブへの直接の API 呼び出しを使用して、Brightcove ブアカウントの動画がリアルタイムに表示され、新しい動画がBrightcove ブにアップロードされたか、AEM DAM での公開プロセスによってメタデータがBrightcove ブに同期されたことを確認できます。

    Brightcove ブからの再生を確認するには、/system/console/ConfigMgr の Brightcove 設定ページで指定されているデフォルトの動画およびプレイリストプレーヤーを使用して、この Brightcove の管理ページで動画をプレビューできます。

    Brightcove ブの管理者ページには、同期データベースボタンも含まれており、Brightcove ブから AEM DAM への同期プロセスを開始します。同期データベースボタンを押して、設定されているすべての Brightcove アカウントを AEM DAM と同期します。  Brightcove ブ内の一部またはすべての動画を同期したくない場合、 Brightcove ブタグが AEM NO_DAM の動画は AEM DAM に同期されません。

    この同期プロセスは、cron ジョブを介して自動化できるため、手動操作は不要です。

    ビデオへのテキストトラックの追加

    現在のところ、テキストトラックを動画に追加できるのは、Brightcove の管理ページ（）だけです/brightcove/admin。テキストトラックは、[ 新しいテキストトラックのアップロード ] ボタンを使用して追加されます。

    DAM でレンディションとしてアップロードされたテキストトラックをサポートすることは、後で実装されるロードマップにあります。

    ポスター画像の更新

    AEM DAM でポスターとサムネイル画像を更新するだけでなく、 次の場所の Brightcove 管理者ページを介してリモート HTTP URL からBrightcove ブにアップロードすることもできます/brightcove/admin。
