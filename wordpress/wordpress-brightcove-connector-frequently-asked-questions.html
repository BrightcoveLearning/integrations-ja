WordPress-Brightcoveコネクタ：よくある質問

このトピックでは、Wordpress-Brightcove コネクタに関するよく寄せられる質問について説明します。

パブリックにアクセスできない WordPress インストールでプラグインを実行できますか？

はい、それは公開されているかどうかにかかわらず機能しますが、ステータス更新、ビデオのアップロードなど、あなたのサイトにコールバックを必要とする機能は利用できなくなります。

 プラグイン/テーマ開発者向けのフィルターはありますか？ 

brightcove_account_actions = [編集、削除];//ブライトコーブのソースを操作する際に使用できるアクションは何ですか。

これはマルチサイトで機能しますか？

はい、それはします。

複数のブライトコーブアカウントを使用できますか？

はい。必要に応じて、多くの Brightcove アカウントからソースを追加できます。

ユーザーに異なる権限を設定できますか？

はいこのドキュメントでは、ユーザー権限のマトリックスが記載されています。ユーザーロールを変更するには、WordPress add_cap 関数を使用することをお勧めします。 https://codex.wordpress.org/add_cap

同期はどのように機能しますか？

このプラグインは、Brightcove API から情報を直接取得して表示し、動画をローカルに同期しません。

アップロードファイルの最大サイズを増やすにはどうすればよいですか？

最大ファイルサイズは、ウェブサーバーと PHP の設定によって決まります。php.ini で upload_max_filesize と post_max_size の値を設定する必要があります。php_ini_loaded_file () は PHP.ini の場所を見つけるのに役立ちます。

変更追跡を有効にして、誰が動画を更新したのか、いつ更新したかを確認するにはどうすればよいですか？

「_change_history」' の内部名を持つ「テキスト」タイプのカスタムフィールドを作成します。動画が更新されるたびに、ユーザ名と現在の時刻がこのフィールドに記録された変更のリストに追加されます。

nginxの場合:
http://nginx.org/en/docs/http/ngx_http_core_module.html#client_max_body_size（client_max_body_size）
Apache の場合:
http://httpd.apache.org/docs/current/mod/core.html#limitrequestbody（LimitRequestBody）
